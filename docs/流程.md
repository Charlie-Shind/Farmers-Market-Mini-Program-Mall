# 农仓平台最终展示流程与验收文档 V2

## 一、文档目标

本文件用于最终项目展示、验收、录屏和问题定位。

本项目不是单一商城页面，而是一个完整农仓平台，包含：

```text
1. C 端用户微信小程序
2. B 端商家微信小程序
3. 平台 Web 管理后台
4. 平台官方商家身份
```

最终展示不应该把所有功能强行串成一条线，而应该采用：

```text
一条主交易闭环
+
四条业务支线
+
一个身份隔离验证
```

这样最稳，也最能体现项目完整度。

------

## 二、最终展示结构

### 1. 主交易闭环

```text
平台配置
→ C端浏览商品
→ C端下单支付
→ B端接单发货
→ C端查看物流
→ C/B聊天
→ C端确认收货
→ C端评价
→ B端回复评价
```

### 2. 业务支线

```text
支线一：售后退款
支线二：优惠券 / 促销价
支线三：限时秒杀
支线四：邻里拼团
```

### 3. 身份隔离验证

```text
普通商家数据
与
平台官方商家数据
必须隔离
```

------

## 三、展示前准备

### 1. 必备账号

| 身份         | 用途                                          |
| ------------ | --------------------------------------------- |
| 普通用户     | C端浏览、下单、聊天、评价、售后               |
| 普通商家     | B端接单、发货、回复、处理售后、提现           |
| 平台管理员   | 后台审核商家、配置 Banner、公告、优惠券、活动 |
| 平台官方商家 | 平台自营商品、平台官方客服、平台自营订单归属  |

------

### 2. 必备基础数据

展示前必须准备：

```text
1. 一个平台管理员账号
2. 一个普通用户账号
3. 一个已审核通过的普通商家账号
4. 一个平台官方商家
5. 至少一个普通商家商品
6. 至少一个平台官方商品
7. 每个商品至少一个 SKU
8. SKU 必须有库存
9. 一个首页 Banner
10. 一条公告消息
11. 一个可领取优惠券
12. 一个秒杀商品
13. 一个拼团商品
14. 一个测试收货地址
15. 一个可用提现账户
```

------

### 3. 展示前必须确认

必须确认以下事项：

```text
1. 后端服务正常启动
2. 小程序连接的是当前后端 API
3. 平台后台连接的是当前后端 API
4. 普通商家 /merchant/* 不会串到平台官方商家
5. C端首页 Banner 不是写死图片
6. B端页面没有 safeGet / safePost 假成功
7. 后台页面没有 catch 后本地假成功
8. 商品、订单、评价、售后、财务都是真实接口数据
```

```text
订单 A：用于完整购买、发货、收货、评价
```

# 第十二章：最终通过标准

最终通过必须满足：

```text
1. 平台配置的数据能影响 C端展示。
2. C端下单的数据能进入 B端订单列表。
3. B端接单发货后，C端订单状态和物流同步变化。
4. C端消息能被 B端收到。
5. B端回复能被 C端看到。
6. C端确认收货后，能提交评价。
7. B端能看到评价并回复。
8. C端申请售后后，B端能处理。
9. B端处理售后后，C端状态同步。
10. 优惠券能真实抵扣订单金额。
11. 秒杀订单使用秒杀价。
12. 拼团订单使用拼团价并绑定 groupBuyId。
13. 商家财务能看到收入、退款、提现。
14. 平台后台能审核商家、查看订单、处理提现。
15. 普通商家不能看到平台官方商家数据。
16. 平台官方商家能处理平台自营订单。
```
