## 3. 小程序用法（推荐）

### 3.1 新代码：必须用 `<wx-icon>` 组件

```json
// 页面 / 组件的 .json
{
  "usingComponents": {
    "wx-icon": "/components/base/wx-icon/wx-icon"
  }
}
```

```xml
<!-- WXML -->
<wx-icon name="home" size="48rpx" color="#2C4A39" />
<wx-icon name="cart" size="40rpx" />
<wx-icon name="warning" size="24rpx" color="#E45D41" />
```

**Props**

| 属性 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `name` | `IconName` | `''` | 必填。从 `IconName` 类型自动补全所有可用图标名 |
| `size` | `string` | `'48rpx'` | 支持 rpx/px/em。例：`'24px'`、`'32rpx'` |
| `color` | `string` | `'#333333'` | 任何 CSS 颜色值（含 `#2C4A39`、`currentColor`、`var(--xxx)`） |
| `strokeWidth` | `number` | `2` | 描边粗细，1-3 视觉效果较佳 |

### 3.2 旧代码：仍可用 `iconPaths` + `<image>`

```ts
// 旧 TS（无需修改即可继续工作）
import { iconPaths } from '../../config/icons';
Page({
  data: { icons: iconPaths },
});
```

```xml
<!-- 旧 WXML（无需修改即可继续工作） -->
<image src="{{icons.home}}" style="width:48rpx;height:48rpx" />
```

**`iconPaths.xxx` 现在返回 SVG data URI 字符串**（不再是本地路径），所以 `<image>` 能直接渲染。**0 迁移成本**。

### 3.3 高级用法：直接拿 SVG 字符串

```ts
import { getIconSvg, getIconPath, type IconName } from '../../config/icons';

// 完整 <svg>...</svg> 字符串（带 viewBox、stroke 等）
const svg = getIconSvg('home');
// '<svg xmlns="..." viewBox="0 0 24 24" ...><path .../></svg>'

// 仅 SVG 内部 body（不含 <svg> 标签）
const body = getIconPath('home');
// '<path .../>'
```

## 4. Admin 后台用法

### 4.1 新代码：用 `<AppIcon>`

```vue
<template>
  <AppIcon name="home" :size="20" color="#2C4A39" label="首页" />
  <AppIcon name="success" :size="24" />
</template>

<script setup lang="ts">
import AppIcon from '@/components/AppIcon.vue';
</script>
```

**Props**（与 `wx-icon` 对齐）

| 属性 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `name` | `string` | 必填 | 任意 `IconName` 字符串 |
| `size` | `number \| string` | `20` | 例：`24`、`'24px'` |
| `color` | `string` | `'currentColor'` | CSS 颜色 |
| `strokeWidth` | `number` | `2` | 描边粗细 |
| `label` | `string` | `''` | 无障碍标签 |

### 4.2 现有 `<AppMessage>` 用法不变

```vue
<AppMessage type="error" title="登录失败" :content="loginError" />
```

`AppMessage` 内部已切换为 `<AppIcon name="messageError">`，业务代码 0 修改。

## 5. 使用次数统计

**设计目标**：知道每个图标在生产中真实使用频次，方便后续按需打包、按需优化、淘汰未使用图标。

### 5.1 自动记录

| 触发点 | 行为 |
|---|---|
| `<wx-icon>` 挂载时 | `recordIconUsage(name)` +1 |
| `<AppIcon>` 渲染时 | `recordIconUsage(name)` +1 |
| 切换 prop | 不重复计数（避免抖动放大） |

### 5.2 查询

```ts
import { dumpIconUsage, getIconUsage, clearIconUsage } from '../../config/icons';

// 当前会话/进程内的统计
const usage = getIconUsage();
// { home: 12, cart: 8, search: 5, ... }

const sorted = dumpIconUsage();
// [{ name: 'home', count: 12 }, { name: 'cart', count: 8 }, ...]

clearIconUsage(); // 重置
```

### 5.3 关闭统计

```ts
import { setIconUsageEnabled } from '../../config/icons';

// 生产环境关闭，减少无意义的函数调用
setIconUsageEnabled(false);
```

> 默认开启。`dumpIconUsage()` 可挂到管理后台"开发工具"页面，方便运营/开发查看。

## 6. 命名规范

| 类别 | 命名 | 例 |
|---|---|---|
| 通用 | kebab-case | `home`, `chevron-right`, `arrow-down` |
| 平台前缀 | `b-` (B端) / `cat-` (分类) / `weather-` (天气) | `b-home`, `cat-fruit`, `weather-sun` |
| 同语义多版本 | 后缀区分 | `message-info`, `message-success` |
| 状态变体 | `order-pending-pay` | `order-pending-pay` / `order-pending-ship` |
| 操作 | 动词原形 | `add` / `remove` / `edit` / `delete` |

> 不再使用 `icon-` 前缀（业内约定俗成）+ 字母数字混排。统一 `kebab-case`。

## 7. 增加新图标的流程

1. **去 https://tabler.io/icons 找图标**（或 https://github.com/tabler/tabler-icons）
2. **打开 SVG 源码**，复制 `<path d="..."/>` 部分
3. **追加到对应分类文件**（`shared/icons/registry/<category>.ts`）
4. **TS 类型自动扩展**（`IconName` 由 `keyof typeof allIcons` 推断）
5. **业务代码直接用** `<wx-icon name="新名字" />` —— IDE 自动补全

```ts
// 例：shared/icons/registry/navigation.ts 新增
export const navigationIcons = {
  home: '...',
  // 新增
  'arrow-up': '<path d="M6 15l6 -6l6 6"/>',
};
```

## 8. 旧页面迁移到组件的指南

### 8.1 何时迁移

| 优先级 | 场景 |
|---|---|
| P0 | 改图标颜色 / 改尺寸 / 改 strokeWidth 的页面 |
| P1 | 数据看板 / 统计页（需要图标使用次数） |
| P2 | 其他普通页面 |

### 8.2 迁移步骤

1. 在页面 .json 中注册 `wx-icon`
2. 把 `<image src="{{icons.xxx}}">` 改为 `<wx-icon name="{{icons.xxx}}">`
3. 验证视觉一致（data URI 和组件应一致）
4. 删除页面中 `data.icons` 字段（可选）
5. 文档标注"已迁移"

### 8.3 不迁移也可以

- 旧 `<image src="{{icons.xxx}}">` 继续工作（用 data URI 渲染）
- 缺点：不能用 strokeWidth 调整描边、不能用 recordIconUsage 统计
- 适用：纯静态展示页面

## 9. 故障排查

| 现象 | 原因 | 解决 |
|---|---|---|
| 图标不显示 | `name` 拼写错 | 检查 `IconName` 类型，IDE 会标红 |
| 颜色不生效 | `<image>` 标签 | 改用 `<wx-icon>` 组件，`<image>` 无法换色 |
| data URI 在 image 标签加载慢 | 大量同图渲染 | 改用 `<wx-icon>` 组件（避免重复解析 data URI） |
| 迁移后视觉错位 | size 单位差异 | 确认是 `rpx` 还是 `px`，`<wx-icon>` 原样透传 |
| TS 报 "iconPaths 不存在" | 旧代码引旧文件 | 重新 `import { iconPaths } from '../../config/icons'`，新 icons.ts 已恢复 |
