# 02-3-接口-平台后台

> 面向平台运营管理员的后台接口说明，覆盖审核、配置、仲裁、看板与基础上传能力。

## 0. 当前实现说明

- 路由前缀统一为 `/api/admin`
- 仅平台管理员角色可访问写接口
- 活动、商品、退款等模块的后台管理能力仍保留在 admin controller 中
- 当前管理端接口主要负责“审核、配置、仲裁、看板、系统维护”

## 1. 管理员登录

```http
POST /api/admin/auth/login
```

- 查询后台账号
- 校验密码哈希
- 签发后台 JWT

## 2. 审核管理

### 2.1 商户审核

```http
POST /api/admin/merchants/{merchantId}/audit
```

- 审核商家入驻资料
- 通过后初始化商家权限和钱包

### 2.2 商品审核

```http
POST /api/admin/products/{productId}/audit
```

- 审核商品详情、SKU、溯源资料
- 拒绝时保留原因

### 2.3 用户管理

```http
GET /api/admin/users
PATCH /api/admin/users/{userId}/status
```

- 支持分页、关键词和状态筛选
- 返回用户昵称、手机号、状态、积分、订单数和最近登录时间
- 敏感字段不对外透出，前端只能依赖后端返回的稳定字段
- 状态变更仅允许 `NORMAL` / `DISABLED`，也兼容数值或下拉枚举透传
- 变更接口返回更新后的状态摘要，不暴露内部实现细节

## 3. 活动与配置

### 3.1 Banner 配置

```http
POST /api/admin/banners
```

**请求体建议：**

```json
{
  "title": "助农专题活动",
  "imageUrl": "https://xxx.com/banner.png",
  "linkType": "activity",
  "linkId": 1001,
  "startAt": "2026-06-09T00:00:00.000Z",
  "endAt": "2026-06-16T00:00:00.000Z",
  "sortOrder": 1,
  "status": "ENABLED"
}
```

- `title` 允许自定义，用于展示“助农专题活动”“新品预售活动”等运营文案
- `imageUrl` 负责主视觉图
- `linkType` / `linkId` 负责落地页跳转
- `startAt` / `endAt` 为空时表示不限制投放时间
- `sortOrder` 决定首页轮播顺序
- `status` 建议使用 `ENABLED` / `DISABLED`

**返回字段建议：**

- `id`
- `title`
- `imageUrl`
- `linkType`
- `linkId`
- `startAt`
- `endAt`
- `sortOrder`
- `status`

### 3.2 活动管理

```http
GET /api/admin/activities
POST /api/admin/activities
GET /api/admin/activities/{activityId}
PUT /api/admin/activities/{activityId}
DELETE /api/admin/activities/{activityId}
POST /api/admin/activities/{activityId}/publish
POST /api/admin/activities/{activityId}/pause
POST /api/admin/activities/{activityId}/finish
POST /api/admin/activities/{activityId}/copy
GET /api/admin/activities/{activityId}/statistics
GET /api/admin/activities/product-candidates
GET /api/admin/activities/drafts
POST /api/admin/activities/drafts
POST /api/admin/activities/drafts/{draftId}/publish
DELETE /api/admin/activities/drafts/{draftId}
```

- 当前活动后台围绕“活动主表 + 活动商品 + 活动规则”展开
- 草稿与正式活动保持分离
- 复制、发布、暂停、结束都保留现有路径

### 3.3 活动商品规则接口建议

```http
GET  /api/admin/activities/{activityId}
POST /api/admin/activities/{activityId}/products
DELETE /api/admin/activities/{activityId}/products/{itemId}
POST /api/admin/activities/{activityId}/rule
PUT  /api/admin/activities/{activityId}/rule
PATCH /api/admin/activities/{activityId}/status
```

- `GET` 用于查看活动详情、商品绑定和规则摘要
- `products` 子资源用于单独维护活动商品，不和主表混写
- `rule` 子资源用于维护秒杀、拼团、满减、折扣的规则体
- `status` 用于草稿、发布、下线、结束等状态流转

## 4. 订单与售后

### 4.1 全量订单查询

```http
GET /api/admin/orders
```

### 4.2 商品审核

```http
POST /api/admin/products/{productId}/audit
```

- 审核商品详情、SKU、溯源资料
- 拒绝时保留原因

### 4.3 商户审核

```http
POST /api/admin/merchants/{merchantId}/audit
```

- 审核商家入驻资料
- 通过后初始化商家权限和钱包

### 4.4 售后仲裁

```http
POST /api/admin/refunds/{refundNo}/arbitrate
```

- 平台介入用户与商家的售后争议

### 4.5 提现管理

```http
GET /api/admin/withdraws
POST /api/admin/withdraws/{applyNo}/audit
```

- 提现列表支持分页、关键词和状态筛选
- 审核接口当前兼容两种请求字段：
  - `action`: `approve` / `reject`
  - `auditStatus`: 兼容前端透传，`3` 表示通过，`4` 表示拒绝
- 建议统一使用 `action` 作为后台审核动作字段，备注可通过 `remark` 传入

## 5. 数据看板

```http
GET /api/admin/dashboard/overview
GET /api/admin/dashboard/sales
GET /api/admin/dashboard/hot-products
GET /api/admin/dashboard/origin-sales
```

- 销量、热销商品、产地产销等统计

## 6. 文件上传

```http
POST /api/files/upload
```

- 统一上传商品图、资质、Banner、售后凭证
- 文件本体存对象存储，数据库只存 URL 或 key
