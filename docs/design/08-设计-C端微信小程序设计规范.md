# 08-设计-C端微信小程序设计规范

> 适用范围：农仓平台 C 端微信小程序的视觉规范、页面结构规范和组件规范。  
> 本文档基于 [`08-设计-总规范.md`](./08-%E8%AE%BE%E8%AE%A1-%E6%80%BB%E8%A7%84%E8%8C%83.md)，用于消费者端页面对齐，不替代交互原型和接口文档。

---

## 1. 文档目标

这份规范的目标是把设计稿中的页面风格、色彩、模块和组件表达成统一标准，方便后续前端实现、设计复盘和页面扩展。

规范重点回答四个问题：

- 页面整体长什么样
- 关键模块如何分区
- 颜色和字号如何统一
- 不同页面如何保持同一套视觉语言

---

## 2. 设计基线

### 2.1 视觉来源

设计稿截图尺寸：`1672 × 941 px`，`RGB` 色彩模式。

当前视觉风格可概括为：

- 自然、温和、偏农产品品牌感
- 以米白和深墨绿为核心
- 强调“产地直供”“新鲜”“可信赖”的感受
- 页面层级清晰，营销区与商品区分区明确

### 2.2 适用页面

- 首页
- 商品分类页
- 商品详情页
- 营销中心页
- 购物车 / 结算页
- 个人中心页

---

## 3. 设计原则

### 3.1 信息优先

页面内容优先体现商品、价格、活动和下单路径，视觉装饰只做辅助。

### 3.2 层级明确

每个页面都应有稳定的层级：

1. 顶部导航或状态区
2. 核心承接区
3. 高频操作区
4. 推荐内容区
5. 底部导航或固定操作栏

### 3.3 统一可复用

分类、标签、卡片、按钮、Banner、推荐位应采用统一组件，避免每个页面重新设计一套样式。

### 3.4 移动优先

设计以手机竖屏为主，宽屏截图仅作为参考展示，不作为实现的唯一基准。

## 4. 色彩规范

### 4.1 核心色板

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 深墨绿 | `#2C4A39` | 主标题、价格、重点信息、主视觉强调 |
| 暖米白 | `#F6F3EB` | 页面背景、浅底色、内容承托 |
| 浅灰 | `#999999` | 次要信息、辅助文案、弱化状态 |
| 深岩灰 | `#333333` | 按钮文字、规格项、正文强调 |
| 米砂金 | `#F8A978` | 会员入口、主按钮、强调标签 |

### 4.2 辅助状态色

如需消息红点、库存告警或异常提示，可使用独立状态色，但不应抢占主视觉。

| 名称 | 建议值 | 用途 |
| --- | --- | --- |
| 提醒红 | `#E45D41` | 消息红点、异常提示、警告状态 |

### 4.3 色彩使用规则

- 主色只用于少量高权重信息，不要铺满大面积正文
- 背景色优先使用暖米白或接近暖米白的浅色系
- 价格、折扣、会员和关键按钮应保持一致的强调方式
- 灰色只用于弱信息，不用于核心操作

## 5. 字体与排版

### 5.1 字体建议

中文环境建议使用系统可用字体栈：

- `PingFang SC`
- `Hiragino Sans GB`
- `Microsoft YaHei`
- `sans-serif`

### 5.2 字号层级

| 层级 | 建议字号 | 场景 |
| --- | --- | --- |
| 页面标题 | 18-20 px | 页面主标题、模块标题 |
| 主信息 | 16-18 px | 商品名、价格、关键活动名 |
| 正文 | 14-15 px | 说明文案、标签说明、列表正文 |
| 辅助文字 | 12-13 px | 副标题、提示信息、说明标签 |

### 5.3 字重建议

- 页面标题：600-700
- 正文：400-500
- 价格与活动标签：600-700
- 次要说明：400

---

## 6. 页面布局规范

### 6.1 页面容器

- 页面主体采用纵向滚动结构
- 内容区块之间保留清晰间距
- 卡片统一圆角，减少硬边界
- 容器建议使用浅色底 + 轻阴影或细边框

### 6.2 间距建议

| 项目 | 建议值 |
| --- | --- |
| 页面左右边距 | 12-16 px |
| 模块间距 | 12-16 px |
| 卡片内边距 | 12-16 px |
| 按钮内边距 | 10-14 px |
| 标签间距 | 6-8 px |

### 6.3 圆角建议

- 主卡片：16-24 px
- 小按钮 / 标签：999px 或 12-16 px
- 图标按钮：圆形

---

## 7. 通用组件规范

### 7.1 按钮

#### 主按钮

- 用途：立即购买、去结算、领取优惠券、提交订单
- 背景：`#2C4A39` 或 `#F8A978`
- 文字：白色或深岩灰
- 圆角：14-18 px

#### 次按钮

- 用途：加入购物车、取消、次级操作
- 背景：浅色
- 边框：浅灰或低饱和绿色
- 文字：`#333333` 或 `#2C4A39`

### 7.2 标签

标签用于表达活动、筛选和状态：

- 秒杀
- 拼团
- 礼盒
- 产地直供
- 有机认证
- 现货 / 预售

标签建议：

- 体积小
- 圆角大
- 颜色统一
- 文案短，不超过 4-6 个汉字为宜

### 7.3 商品卡片

商品卡片建议包含：

- 商品图
- 商品标题
- 价格
- 简要标签
- 次级说明

卡片原则：

- 图片优先
- 价格突出
- 信息不要过密
- 同屏卡片保持一致高度优先

### 7.4 Banner

Banner 用于承接活动和品牌主张：

- 首页首屏必须清晰
- 文案短，突出“产地直供”“限时秒杀”“新鲜直达”
- 图文一体，不要同时塞入过多卖点
- Banner 下方可加标签增强识别

### 7.5 图标

图标建议使用线性风格，统一描边粗细和圆角。

- 首页快捷入口：线性图标
- 底部导航：简洁图标
- 个人中心功能列表：图标 + 文本

## 8. 页面规范

### 8.1 首页

首页是首屏转化核心，结构建议如下：

1. 顶部状态栏
2. 定位、搜索、消息入口
3. 主 Banner / 产地直供宣传图
4. 快捷入口四宫格
5. 商品分类模块
6. 智能推荐 / 热销榜单
7. 底部导航

首页视觉要点：

- 背景偏暖米白
- 主色使用深墨绿
- 首屏要先给“新鲜直达”的品牌感
- 活动入口要明显，但不能喧宾夺主

### 8.2 商品分类页

分类页目标是让用户快速找到品类。

建议模块：

- 一级类目入口
- 筛选标签
- 商品列表

类别建议：

- 时令水果
- 粮油干货
- 肉禽蛋奶
- 特产礼盒

### 8.3 商品详情页

详情页目标是促成购买决策，建议模块如下：

- 商品图片 / 视频轮播
- 商品标题
- 商品价格
- 规格选择
- 收藏 / 喜欢
- 服务说明
- 图文详情
- 底部购买栏

视觉要求：

- 价格必须最醒目
- 规格项要有明确选中态
- 收藏等次级动作使用浅灰
- 立即购买与加入购物车必须有明确层级

### 8.4 营销中心页

营销中心承接促销和裂变活动，建议模块如下：

- 新人专享券
- 限时秒杀
- 拼团活动
- 积分兑换
- 活动商品列表

视觉要求：

- 优惠标签统一使用深墨绿或米砂金
- 倒计时和折扣信息要突出
- 活动入口保持清晰，不要堆叠过多信息

### 8.5 购物车 / 结算页

购物车和结算页负责完成下单闭环，建议模块如下：

- 商品勾选列表
- 数量调整
- 优惠券抵扣
- 积分抵扣
- 配送方式
- 收货地址
- 合计金额
- 去结算按钮

视觉要求：

- 总价信息必须固定且醒目
- 主按钮应放在底部固定区
- 推荐商品可存在，但不能干扰结算主流程

### 8.6 个人中心页

个人中心负责资产、订单和服务入口，建议模块如下：

- 用户头像
- 成长值 / 积分
- 会员中心入口
- 全部订单
- 地址管理
- 卡券包
- 收藏商品
- 关注商品
- 客服入口
- 溯源码 / 二维码查询

视觉要求：

- 会员入口使用米砂金强调
- 订单和资产信息使用深墨绿
- 功能列表保持直排、简单、稳定

## 11. C 端结论

C 端设计的核心是“让用户快速看懂商品并完成下单”。

页面设计优先级：

1. 商品和价格清楚
2. 活动和优惠清楚
3. 购买路径清楚
4. 个人资产入口清楚

## 14. 开发对齐检查项

实现页面时建议检查以下内容：

- 页面主色是否统一
- 价格、按钮、标签是否使用一致的强调色
- 分类、筛选、卡片是否复用同一套组件
- 间距是否保持稳定
- 首页首屏是否足够明确
- 详情页购买按钮是否明显
- 购物车结算路径是否清晰
