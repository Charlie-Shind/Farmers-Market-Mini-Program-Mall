# 08-设计-总规范

> 这是农仓项目的顶层设计规范，负责统一视觉语言、组件规则和分层方式。  
> 下挂三个分支文档：`C 端微信小程序`、`B 端微信小程序`、`网页后台`。  
> B 端和 C 端都属于微信小程序，只是页面结构和业务入口不同。

---

## 1. 规范目标

总规范只回答一件事：整个农仓产品体系应该长什么样、怎么统一。

它不直接描述某一个页面，而是定义：

- 全局色板
- 字体与间距
- 按钮、卡片、标签等基础组件
- 小程序与网页后台的分层边界
- B/C 共用的视觉基座

---

## 2. 平台分层

### 2.1 C 端微信小程序

面向普通消费者，重点是浏览、筛选、下单、支付和个人资产管理。

### 2.2 B 端微信小程序

面向商家或农户，重点是店铺管理、商品管理、订单履约和经营看板。

### 2.3 网页后台

面向平台管理员，重点是审核、配置、监管、数据看板和系统级管理。

---

## 3. 全局设计原则

1. 信息优先，装饰克制
2. 视觉统一，组件复用
3. 移动优先，后台响应式
4. 关键操作必须高可见
5. 价格、活动、状态提示要有一致的强调方式

---

## 4. 全局色板

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 深墨绿 | `#2C4A39` | 主标题、价格、关键强调 |
| 暖米白 | `#F6F3EB` | 页面背景、浅色底 |
| 浅灰 | `#999999` | 辅助文案、次要状态 |
| 深岩灰 | `#333333` | 正文、规格、按钮文字 |
| 米砂金 | `#F8A978` | 主按钮、会员、优惠强调 |
| 提醒红 | `#E45D41` | 红点、异常、提醒 |

使用约束：

- 深墨绿只用于高权重信息
- 米砂金用于激励和转化，不用于大段正文
- 浅灰只用于辅助说明

---

## 5. 字体与间距

### 5.1 字体栈

- `PingFang SC`
- `Hiragino Sans GB`
- `Microsoft YaHei`
- `sans-serif`

### 5.2 字号层级

| 层级 | 建议字号 |
| --- | --- |
| 页面主标题 | 18-20 px |
| 商品名 / 关键数据 | 16-18 px |
| 正文 | 14-15 px |
| 辅助说明 | 12-13 px |

### 5.3 间距建议

| 项目 | 建议值 |
| --- | --- |
| 页面左右边距 | 12-16 px |
| 模块间距 | 12-16 px |
| 卡片内边距 | 12-16 px |
| 标签间距 | 6-8 px |

---

## 6. 通用组件

### 6.1 按钮

- 主按钮：深墨绿或米砂金底
- 次按钮：浅底 + 细边框
- 危险按钮：少量使用提醒红

### 6.2 标签

适合表达：

- 秒杀
- 拼团
- 礼盒
- 产地直供
- 有机认证
- 现货 / 预售

### 6.3 卡片

统一原则：

- 圆角统一
- 图文比例稳定
- 价格高亮
- 文字密度控制

### 6.4 导航

- C/B 端小程序：底部导航简洁、低干扰
- 网页后台：左侧导航或顶部导航，强调功能分组

---

## 7. B/C 共用规范

B 端和 C 端虽然页面不同，但共用同一套小程序视觉基座：

- 统一色板
- 统一圆角和阴影层级
- 统一按钮语义
- 统一卡片语言
- 统一图标风格

差异只体现在：

- 页面信息密度
- 核心转化路径
- 入口文案
- 功能复杂度

---

## 8. 后台规范

网页后台不追求小程序的温和感，而是更偏：

- 高密度信息展示
- 更强的表格和列表能力
- 更清晰的层级分组
- 更重的管理效率

后台仍然沿用总规范中的色板，但布局和交互可以更偏管理型界面。

## 10. 结论

以后新增设计内容时，优先判断它属于哪一层：

- 全局规则，写到 `总规范`
- 消费者页面，写到 `C 端规范`
- 商家页面，写到 `B 端规范`
- 平台管理页面，写到 `网页后台规范`
