# 23. 定位选择页

## 页面定位

- 作为首页「定位」入口的独立页面
- 首页不改版，只调整「定位」入口的跳转行为
- 提供自动定位和手动选点两种方式
- 选中的定位会同步到首页显示

## 设计目标

- 只保留定位相关信息，不承载其它业务入口
- 页面默认进入自动定位流程
- 手动选点通过地图拖动完成，地图中心点即为选点结果
- 仍保留微信地图搜索入口作为补充能力
- 让用户明确知道当前选中的位置、坐标和来源

## 页面结构

1. 顶部导航
- 返回
- 页面标题「定位选择」

2. 定位概览
- 当前定位名称
- 地址说明
- 经纬度信息
- 当前定位方式

3. 操作区
- 页面进入时自动获取当前位置
- 手动搜索选点
- 确认并返回

4. 地图区域
- 地图显示当前定位点
- 地图中心点作为选点结果
- 支持拖动地图更新位置

5. 辅助提示
- 当前来源说明
- 自动定位与手动选点的差异说明
- 权限不足时的引导提示

## 页面状态

1. 默认态
- 读取本地已选定位
- 没有缓存时显示默认城市名与占位地址

2. 自动定位态
- 使用 `wx.getLocation`
- 页面打开后自动刷新名称、地址和经纬度
- 失败时提示需要开启定位权限

3. 手动选点态
- 使用 `map` 组件拖动地图
- 通过 `getCenterLocation` 读取地图中心点
- 如需搜索地址，可继续调用 `wx.chooseLocation`
- 成功后刷新名称、地址和经纬度
- 失败时提示重新选择

4. 权限缺失态
- 引导用户去系统设置打开 `scope.userLocation`
- 不阻断返回操作

## 官方接口

- `wx.getLocation`
- `wx.chooseLocation`
- `wx.openLocation`
- `map`

## 数据约定

- `source`
  - `auto`：自动定位
  - `manual`：手动选点
- `name`
  - 定位名称
- `address`
  - 定位地址
- `latitude`
  - 纬度，数字类型
- `longitude`
  - 经度，数字类型
- `updatedAt`
  - 最近更新时间

## 交互规则

- 页面打开后自动调用系统定位
- 手动入口优先通过拖动地图选点
- 搜索地址作为补充能力，不作为唯一方式
- 选点后立即更新页面信息和本地存储
- 首页读取最新定位名称作为顶部显示
- 确认按钮只负责返回上一页，不额外修改页面结构

## 视觉说明

- 保持首页同款农仓绿色调
- 卡片化信息区，突出当前定位信息
- 地图区域作为页面主视觉
- 返回按钮使用纯文字，不使用图标
- 页面不显示全屏加载遮罩
- 页面只保留必要操作，不增加附加业务入口
