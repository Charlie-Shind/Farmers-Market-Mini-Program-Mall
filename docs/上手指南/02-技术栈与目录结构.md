> 看清三个端用什么技术、目录怎么组织、关键文件在哪。

## 一、技术栈总览

| 层 | 选型 | 版本（来自 package.json） | 用途 |
|---|---|---|---|
| 前端框架（Web） | Vue | 3.5.13 | 管理后台 SPA |
| 构建（Web） | Vite | 7.1.0 | 开发服务器与生产构建 |
| 类型系统 | TypeScript | 5.9.x | 前后端统一类型 |
| 后端框架 | NestJS | 11.x | 路由、依赖注入、Guard |
| ORM | Prisma | 7.x | 数据库访问、迁移 |
| 数据库 | PostgreSQL | — | 业务数据持久化 |
| 缓存 | Redis | — | Token、库存锁、订单超时、缓存 |
| 对象存储 | MinIO | —（与 S3 兼容） | 图片 / 视频 / 资质文件 |
| 鉴权 | Passport JWT | — | Bearer Token 鉴权 |
| 校验 | class-validator / class-transformer | — | DTO 入参校验 |
| 小程序 | 微信原生 + TS + Less | 基础库 2.32.3 | C 端 / B 端两个端共用 |
| 部署 | Linux + Nginx + PM2 / Docker | — | 见 `12-基础设施-Docker编排.md` |

支付通道为**微信支付 V3**，文件存储后续可切换到 COS / OSS / 微信云存储。

## 三、后端 `backend/`

### 3.1 目录结构

```text
backend/
├── prisma/
│   └── schema.prisma         数据库 schema，模型变更的单一来源
├── prisma.config.ts          Prisma 启动配置（指向 .env.local / .env）
├── src/
│   ├── main.ts               应用入口，注册全局 Pipe / Filter / 前缀
│   ├── app.module.ts         顶层模块组合
│   ├── app.controller.ts     根路由（健康检查）
│   ├── app.service.ts
│   ├── common/               通用能力
│   │   ├── common.module.ts  全局模块：Prisma / Guard / Interceptor / Filter
│   │   ├── constants/        常量（角色、Token 类型等）
│   │   ├── decorators/       @Public() / @Roles() / @CurrentUser()
│   │   ├── enums/            状态枚举
│   │   ├── filters/          全局异常过滤器（统一错误信封）
│   │   ├── guards/           JwtAuthGuard / RolesGuard
│   │   ├── interceptors/     ResponseInterceptor（统一成功信封）
│   │   ├── prisma/           PrismaService + PrismaModule
│   │   ├── services/         跨域服务（如 platform-data.service.ts）
│   │   ├── storage/          MinIO 客户端
│   │   ├── types/            共享类型
│   │   └── utils/
│   └── modules/
│       ├── identity/         鉴权与用户
│       │   ├── auth/         /api/identity/auth/* （status / anonymous / me）
│       │   ├── users/
│       │   └── roles/
│       └── business/         业务域
│           ├── address/      收货地址
│           ├── admin/        /api/admin/* 后台路由
│           ├── cart/         /api/app/cart
│           ├── file/         /api/files/upload
│           ├── home/         /api/app/home 首页聚合
│           ├── marketing/    营销活动 / 卡券
│           ├── merchant/     /api/merchant/* 商家路由
│           ├── message/      系统消息
│           ├── order/        /api/app/orders /api/merchant/orders
│           ├── payment/      微信支付 + 回调
│           ├── product/      /api/app/products /api/merchant/products
│           ├── qrcode/       小程序码相关
│           ├── refund/       售后
│           └── business.module.ts
├── test/                     测试样例
├── uploads/                  本地上传兜底目录（启用 MinIO 时可忽略）
├── docs/                     后端专属子文档
├── .env.example              环境变量样例
├── .env.local                本地环境（不入 git）
├── package.json              脚本：start:dev / start:clean / build 等
└── nest-cli.json
```

### 3.2 启动脚本

| 脚本 | 作用 |
|---|---|
| `npm run start:dev` | `nest start --watch`，开发首选 |
| `npm run start` | 跑 `dist/main.js`，需要先 `build` |
| `npm run start:clean` | 清掉 `dist/` 与 tsbuildinfo 后再 build + 启动 |
| `npm run build` | `nest build` |
| `npm run prisma:generate` | 拉取 `schema.prisma` 生成 client |
| `npm run prisma:studio` | 打开 Prisma Studio 看数据 |

### 3.3 关键模块清单

- **基础**：NestJS 11 + Prisma 7 + PostgreSQL + Redis + MinIO
- **统一响应**：`ResponseInterceptor` 包装为 `{ success, statusCode, path, timestamp, data }`
- **统一错误**：`HttpExceptionFilter` 包装为 `{ success, statusCode, path, timestamp, message, error }`
- **鉴权**：`JwtAuthGuard` + `RolesGuard`，通过 `@Public()` `@Roles(...)` 控制
- **种子数据**：`common/services/platform-data.service.ts` 提供 mock 数据，让前后端在没有完整业务逻辑时也能联调

## 四、基础设施与本地端口

| 服务 | 端口 | 启动方式 (推荐在 base 目录下运行) |
|---|---|---|
| PostgreSQL | `6001` | `docker compose --env-file ../.env.local up -d postgres` |
| Redis | `6003` | `docker compose --env-file ../.env.local up -d redis` |
| MinIO API | `6004` | `docker compose --env-file ../.env.local up -d minio` |
| MinIO Console | `6005` | 同上 |
| MinIO UI（opens3/console） | `6006` | `docker compose --env-file ../.env.local --profile tools up -d` |
| 后端 API | `6002` | `cd backend && npm run start:dev` |
| 管理后台 | `6007` | `cd admin && npm run dev`（`/api` 代理到 6002） |

> MinIO 在 `.env.local` 通过 `MINIO_REQUIRED=false` 可以不强制启用，便于无对象存储场景。

```text
admin/
├── index.html
├── vite.config.ts         端口 6007，/api 代理到 6002
├── tsconfig.json
├── src/
│   ├── main.ts
│   ├── App.vue

│   ├── styles.css

- Vue 3.5 + `<script setup>` + Composition API

cd admin

npm run dev
# 浏览器 http://127.0.0.1:6007/admin/
```

### 6.2 启动

1. 安装「[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)」
2. 工具 → 导入项目
   - 项目目录：`wxapp/`
   - AppID：已写入 `project.config.json`（`wx66f19691380795b8`），如需更换请同步修改
3. 编译即可在模拟器预览

> 模拟器无法访问 `localhost`，后端调试时需使用局域网 IP 或开启「不校验合法域名」。
