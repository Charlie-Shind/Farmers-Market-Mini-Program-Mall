# 03 · 本地开发环境搭建

> 把后端、管理后台、小程序三端都跑起来。预计耗时 20–40 分钟（取决于网络与 Docker 镜像拉取速度）。

---

## 0. 准备清单

| 工具 | 最低版本 | 备注 |
|---|---|---|
| Node.js | 20 LTS | `node -v` 验证；`backend/package.json` 写的是 `@types/node ^25`，可向下兼容 |
| npm | 10+ | 随 Node 20 自带 |
| Docker Desktop | 任意较新 | 用于启动 PostgreSQL / Redis / MinIO |
| 微信开发者工具 | 最新稳定版 | 用于打开小程序 |
| Git | 任意 | 拉仓库 |

> Windows 用户请使用 **PowerShell** 或 **Git Bash**；下文命令按 Unix shell 风格书写。

## 1. 启动基础设施（PostgreSQL / Redis / MinIO）

基础设施由 `base/docker-compose.yml` 配置文件拉起。推荐使用根目录下的 `.env.local` 环境变量配置文件：

```bash
# 在仓库根目录运行，指定配置文件与环境变量：
docker compose -f base/docker-compose.yml --env-file .env.local up -d

# 或者进入 base 目录运行：
cd base
docker compose --env-file ../.env.local up -d
```

启动后可用以下命令确认状态（以在 base 目录运行为例）：

```bash
docker compose ps
docker compose logs -f postgres
docker compose logs -f redis
docker compose logs -f minio
```

如需 MinIO Console / `mc` 工具容器：

```bash
docker compose --profile tools up -d
```

### 1.1 端口速查

| 服务 | 端口 |
|---|---|
| PostgreSQL | `6001` |
| Redis | `6003` |
| MinIO API | `6004` |
| MinIO Console | `6005` |
| MinIO UI（opens3/console） | `6006` |

### 1.2 默认账号

| 服务 | 用户 | 密码 |
|---|---|---|
| PostgreSQL | `farm` | `change-me` |
| Redis | — | `change-me` |
| MinIO | `minioadmin` | `change-me` |

> **强烈建议**：把 `.env.example` 复制为根目录 `.env` 后立即修改所有 `change-me`。

## 2. 启动后端（`backend/`）

### 2.1 准备环境变量

```bash
cd backend
cp .env.example .env.local
```

`.env.local` 中**必须**修改的项：

- `JWT_SECRET`：替换为长随机字符串
- `DATABASE_URL` / `REDIS_URL`：与本地基础设施账号一致
- `WECHAT_MINI_PROGRAM_APP_ID` / `WECHAT_MINI_PROGRAM_SECRET`：暂时可用占位值，不影响本地业务
- `MINIO_*`：默认配置可工作；如不想用 MinIO，设 `MINIO_REQUIRED=false`

### 2.2 安装依赖与生成 Prisma Client

```bash
npm install
npm run prisma:generate
```

### 2.3 初始化数据库表结构

```bash
npx prisma migrate dev
# 或
npx prisma db push
```

> 第一次拉仓库时 `prisma/migrations/` 可能为空，看仓库当前情况选择 `migrate dev` 还是 `db push`。

### 2.4 启动开发服务器

```bash
npm run start:dev
# 或：清掉 dist 后再 build + start
# npm run start:clean
```

成功后控制台会显示：

```text
[Nest] LOG [NestFactory] Starting Nest application...
[InstanceLoader] AppModule dependencies initialized
[InstanceLoader] PrismaModule dependencies initialized
...
Application listening on http://localhost:6002
```

打开浏览器访问 `http://localhost:6002/api/identity/auth/status` 应返回统一响应信封。

### 2.5 验证

```bash
# 1. 鉴权状态（公开）
curl http://localhost:6002/api/identity/auth/status

# 2. 申请访客 token
curl http://localhost:6002/api/identity/auth/anonymous

# 3. 用访客 token 调商品列表
curl -H "Authorization: Bearer <token>" http://localhost:6002/api/app/products
```

### 2.6 常用脚本

| 命令 | 说明 |
|---|---|
| `npm run start:dev` | 开发热重载（推荐） |
| `npm run start` | 跑 `dist/main.js`（需要先 `build`） |
| `npm run start:clean` | 强制清缓存后构建并启动 |
| `npm run build` | 仅构建 |
| `npm run prisma:studio` | 打开 Prisma Studio 看数据 |
| `npm run prisma:generate` | 重新生成 client |

## 3. 启动管理后台（`admin/`）

```bash
cd admin
npm install
npm run dev
```

Vite 启动后：

- 本地地址：`http://127.0.0.1:6007/admin/`
- `/api` 请求已通过 Vite proxy 转发到 `http://127.0.0.1:6002`

### 3.1 常见操作

| 命令 | 说明 |
|---|---|
| `npm run dev` | 启动开发服务器 |
| `npm run build` | 生产构建（`vue-tsc -b && vite build`） |
| `npm run check` | 仅做类型检查 |
| `npm run preview` | 预览生产构建 |

### 3.2 启动后台后端进程

管理后台的部分页面（如登录）需要后端 `POST /api/admin/auth/login`。确认后端已经跑起来，否则登录会报网络错误。

## 4. 启动小程序（`wxapp/`）

小程序无需 Node 启动，但要靠微信开发者工具加载。

### 4.1 准备工作

- 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 用微信扫码登录工具

### 4.2 导入项目

1. 工具 → 导入项目
2. **项目目录**：仓库内的 `wxapp/`
3. **AppID**：`wxapp/project.config.json` 中已写好 `wx66f19691380795b8`；如需更换请同步修改
4. **项目名称**：自定义

### 4.3 配置后端地址

打开 `wxapp/miniprogram/config/env.ts`，按当前环境修改 `baseURL`：

- 本地联调：使用局域网 IP，例如 `http://192.168.1.10:6002`
- 真机调试：同样使用局域网 IP，并确保「微信开发者工具」勾选「不校验合法域名」

### 4.4 编译与预览

导入后开发者工具会自动编译：

- **模拟器**：内置基础库 2.32.3，可直接预览
- **真机调试**：扫码即可在手机上跑

### 4.5 常见问题

| 现象 | 排查 |
|---|---|
| `request:fail` | 后端未启动 / 端口不通 / 没勾选「不校验合法域名」 |
| 登录后无数据 | `wxapp/miniprogram/config/env.ts` 的 baseURL 与后端不一致 |
| 真机白屏 | 关闭代理 / 检查手机与电脑是否在同一局域网 |

## 5. 三端联调最短路径（TL;DR）

```bash
# 终端 1：基础设施
docker compose -f base/docker-compose.yml --env-file .env.local up -d

# 终端 2：后端
cd backend
cp .env.example .env.local   # 至少改 JWT_SECRET
npm install
npm run prisma:generate
npx prisma db push           # 初始化表
npm run start:dev

# 终端 3：管理后台
cd admin
npm install
npm run dev
# 浏览器打开 http://127.0.0.1:6007/admin/

# 微信开发者工具
# 导入 wxapp/，编译预览
```

跑通以上四步后，你应该能：

- 在管理后台看到种子数据
- 在小程序模拟器看到首页与商品列表
- 通过后端 `curl` 拉到一致的接口响应
