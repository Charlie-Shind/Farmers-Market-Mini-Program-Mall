# 01 · 项目概览

> 5 分钟搞清楚：这是谁的项目、为谁服务、做到哪一步了。

---

## 1. 一句话定位

**浔源农仓** 是一个面向农产品的多端电商平台，由后端 API、管理后台、C 端用户小程序、B 端商家小程序四个交付物组成，当前已搭建完整骨架并正在补齐端到端交易闭环。

补充说明：

- 当前不对外开放独立 B 端商家运营入口，平台商品默认归属官方账号
- 官方账号名称默认是 `浔源农仓`，后续可在后台系统配置里修改
- 客服入口和平台商品管理会统一指向这个官方账号，避免多身份分散

---

## 2. 服务对象

平台服务三类用户，对应三种入口：

| 角色 | 入口 | 关注的核心问题 |
|---|---|---|
| 普通消费者（C 端） | 微信小程序 | 浏览商品、下单支付、查看订单与售后、领券积分 |
| 农户 / 商家（B 端） | 微信小程序 | 入驻开店、维护商品、处理订单与发货、提现 |
| 平台运营（管理方） | Web 管理后台 | 审核商家与商品、配置 Banner / 活动 / 类目、仲裁售后、看数据看板 |

详细需求基线见 [`01-需求-农仓平台.md`](../01-%E9%9C%80%E6%B1%82-%E5%86%9C%E4%BB%93%E5%B9%B3%E5%8F%B0.md)。

## 6. 角色与权限模型

后端用 NestJS Guard + JWT 做鉴权，所有用户都属于以下四种角色之一：

| 角色 | 含义 |
|---|---|
| `USER` | 已登录的 C 端消费者 |
| `MERCHANT` | 已入驻的 B 端商家 |
| `ADMIN` | 平台运营 |
| `GUEST` | 未登录的访客，由后端颁发临时 token |

Token 分为两种类型：

- `access` — 正式登录态
- `guest` — 访客态，用于浏览 / 临时购物车

路由层装饰器约定：

- `@Public()` 显式声明该路由不需要鉴权
- `@Roles('USER', 'MERCHANT', ...)` 限定角色

## 7. 术语速查

| 术语 | 含义 |
|---|---|
| SKU | 商品的规格组合（如「5 斤装 / 礼盒装」） |
| 拆单 | 一个用户订单按商家拆成多个商家订单履约 |
| 团长 | 社区分销节点，负责拉新与自提（扩展模块，未上线） |
| Banner | 首页顶部运营活动轮播位 |
| 溯源 | 农产品产地 / 资质 / 检测报告追溯信息 |
