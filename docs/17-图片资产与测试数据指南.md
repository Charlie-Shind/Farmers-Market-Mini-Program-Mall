# 17-图片资产与测试数据指南

本文档记录了浔源农仓平台项目中的测试图片资产（Mock Images）体系。这些资产存放于根目录下的 `mock_images/` 文件夹中，用于本地及测试环境下的 **API 上传接口测试**、**数据库 Seed 数据初始化**。当前约定是：开发/测试环境可以使用这些素材完成对象存储上传和种子数据恢复，生产环境不应再依赖 mock 注入。

---

## 1. 资产设计背景与必要性

* **现状缺口**：
  * 项目的前端图标（`wxapp`）均已采用 SVG 路径和 Icon 注册系统进行开发，无需额外下载位图资产。
  * 数据库及 API（如商品封面、商户 Logo、退款证明、二维码卡片）存在大量图片 URL 字段需求，必须落到对象存储真实地址，不能再继续返回假 CDN 链接。
  * 小程序和后台的默认示例数据只允许出现在开发/测试环境，生产环境需要通过真实上传和真实落库使用。
* **解决方案**：
  * 在根目录下统一建立 [mock_images/](../mock_images) 文件夹，存放 7 张符合绿色、有机、质感农仓设计调性的高清 PNG 图片，作为 seed 上传源和接口联调素材。

## 3. 快速 API 上传联调指南

后端接口提供了通用的静态文件上传服务：`POST /files/upload`，多部分表单字段名指定为 `file`。

开发人员可以使用以下命令，快速将 `mock_images/` 下的资产推送到测试服务器中以获取在线 URL。

### 3.1 PowerShell (Windows 开发者环境)
```powershell
# 上传农夫头像
Invoke-RestMethod -Uri "http://localhost:3000/files/upload" `
  -Method Post `
  -Form @{ file = Get-Item "..\mock_images\avatar_farmer.png" }

# 上传营业执照以作入驻申请
Invoke-RestMethod -Uri "http://localhost:3000/files/upload" `
  -Method Post `
  -Form @{ file = Get-Item "..\mock_images\qual_business_license.png" }
```

### 3.2 curl 命令行 (Git Bash / Linux / macOS)
```bash
# 上传商品图片（脐橙）
curl -X POST http://localhost:3000/files/upload \
  -F "file=@../mock_images/product_orange.png"

# 上传售后损坏照片
curl -X POST http://localhost:3000/files/upload \
  -F "file=@../mock_images/refund_damaged_box.png"
```

## 4. 数据库 Seed 与 Mock 初始化建议

在编写 `backend/prisma/seed.ts` 或后端 demo seed 逻辑时，统一采用下面的方式：

1. **先上传，再落库**：
   通过 `POST /api/files/upload` 或后端对象存储服务将 `mock_images/` 上传到 MinIO，拿到真实公开 URL 后再写入 `Product`、`Banner`、`Merchant`、`Message` 等表。
2. **只在开发/测试环境注入 demo 数据**：
   后端当前已经把 demo seed 做了环境开关控制，生产环境不会自动注入 mock 数据。
3. **二维码与分享卡片也必须是真实文件**：
   由后端动态生成的分享图、二维码图同样上传到对象存储，不允许继续返回外部假域名或临时本地路径。
