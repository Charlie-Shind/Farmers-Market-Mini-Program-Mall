# 01-需求-农仓平台

> 原始需求基线，后续补充默认以此为准。

## 一、C 端用户微信小程序客户端需求（面向普通消费者，理解为前台用户下单主流程）

### 1.1 首页模块（首屏承接活动、检索和转化）

- 顶部功能：用户自动 / 手动切换定位、全站商品搜索框、系统消息中心（活动推送、订单通知）（我们理解为：首页顶部固定工具条，负责定位、搜索、消息提醒三件事）
- Banner 位：展示助农专题活动、新品预售活动（我们理解为：首屏主视觉活动区，优先展示转化型活动）
- 快捷入口：秒杀专区、拼团专区、礼盒专区、产地直供专区 4 个快捷导航（我们理解为：4 个高频业务入口，放在 Banner 下方）
- 内容推荐：智能品类推荐、平台热销商品榜单（我们理解为：下滑后的推荐流和榜单区，用来承接继续浏览）

### 1.2 商品分类模块（帮助用户快速找品类）

- 基础类目：时令果蔬、肉禽蛋奶、粮油干货、特产礼盒四大分类（我们理解为：一级类目入口）
- 筛选标签：有机认证、产地筛选、现货商品、预售商品 4 种筛选条件（我们理解为：列表页/推荐页可复用的筛选条件）

### 1.3 商品详情模块（商品购买前的核心决策页）

- 商品展示：图文详情、田间实拍短视频、多规格定价、溯源码扫码查看（我们理解为：详情页的内容展示层）
- 服务说明：坏果包赔保障、冷链直发服务说明（我们理解为：降低购买顾虑的保障说明）
- 操作功能：加入购物车、立即购买、商品收藏、在线对接商家客服（我们理解为：详情页底部操作区）

### 1.4 营销中心模块（提升转化和复购）

- 优惠活动：新人优惠券领取、满减活动、限时秒杀、邻里拼团（我们理解为：营销活动入口和活动承接页）
- 积分体系：下单获得购物积分、积分兑换商品 / 优惠券（我们理解为：复购激励和积分账户体系）

### 1.5 购物车 & 结算模块（完成下单闭环）

- 购物车：商品勾选、修改购买数量、凑单优惠提示（我们理解为：下单前的集中确认页）
- 订单结算：收货地址选择、配送方式（快递 / 同城配送 / 门店自提）、优惠抵扣（券 + 积分）、微信支付（我们理解为：提交订单的最终确认页）

### 1.6 个人中心模块（用户资产和订单管理）

- 订单业务：全状态订单查询（待付 / 待发 / 待收 / 已完成）、售后退款申请（我们理解为：订单状态查询和售后入口）
- 资产信息：收货地址管理、卡券包、积分明细、收藏商品列表（我们理解为：用户资产中心）
- 服务：在线平台客服咨询（我们理解为：统一客服入口）

## 二、B 端农户 / 商家入驻微信小程序端需求（商户后台，理解为商家经营后台）

> 说明：当前项目未对外开放独立 B 端商家运营入口，平台默认使用官方账号承接商品、客服与平台商品归属；若后续开放 B 端，再拆分真实商家流程。

### 2.1 店铺设置（商家入驻与基础配置）

- 资质管理：营业执照等资质认证、店铺基础资料填写、农产品产地资质上传提交审核（我们理解为：商家入驻审核前置资料）

### 2.2 商品管理（商家维护商品上下架）

- 商品上下架操作、库存增减维护、商品规格编辑、溯源资料录入绑定（我们理解为：商品日常经营后台）

### 2.3 订单处理（商家发货与售后处理）

- 新订单接单、订单发货操作、用户售后订单处理（我们理解为：商家订单履约流程）

### 2.4 财务管理（商家收入和提现）

- 店铺营收明细查看、平台佣金扣费明细、账户余额提现申请（我们理解为：收入、扣费、提现三个看板）

### 2.5 自营营销（商家自己做活动）

- 单品自主折扣设置、商家自建拼团活动、店铺专属优惠券创建发放（我们理解为：门店级活动能力）

## 三、平台 WEB 管理后台需求（平台运营管理员，理解为平台级监管与配置）

### 3.1 用户管理（全平台用户治理）

- 全平台会员信息管理、用户黑名单管控、用户消费数据统计查看（我们理解为：用户生命周期和风险控制）

### 3.2 商户管理（商家准入与监管）

- 农户 / 商家入驻资料审核、平台统一佣金规则配置、违规店铺启用 / 停用（我们理解为：平台审核、规则配置和风控处置）

### 3.3 商品管理（平台商品治理）

- 平台类目新增 / 编辑配置、商家上架商品审核、违规商品强制下架（我们理解为：类目和商品审核中心）

### 3.4 活动管理（首页和平台活动统一配置）

- 首页 Banner 内容配置、平台官方统一活动创建与配置（我们理解为：首屏活动内容和运营活动编辑器）

### 3.5 订单 & 售后（平台仲裁和售后监管）

- 全平台订单汇总查询、售后纠纷平台仲裁、用户赔付申请审核（我们理解为：订单总览和售后仲裁工作台）

### 3.6 数据看板（经营分析）

- 平台整体销量报表、访客流量数据、热销商品排行、产地产销统计报表（我们理解为：运营 BI 看板）

### 3.7 物流配置（配送规则基础设施）

- 全局运费模板配置、配送区域划分、社区自提点新增与管理（我们理解为：配送能力和自提点配置中心）

## 四、特色增值模块需求（平台差异化业务，理解为后续可选扩展能力）

- 社区团长分销：团长分销链路搭建、自提订单自动分佣结算（我们理解为：社区推广和裂变分佣能力）
