# 13-开发任务计划

> 目标：把当前项目按“最小可用闭环”推进，而不是一次性铺满全部扩展功能。  
> 范围：微信小程序、后端身份、后台管理、基础环境与日志。

---

## 1. 当前阶段目标

当前阶段只做四件事：

- 微信小程序登录 / 注册合一
- 后台管理面板登录
- 基础组件与基础请求层
- 环境区分与日志分级

---

## 2. 开发优先级

### P0：必须先完成

1. 微信手机号授权登录与注册
2. 后端身份接口打通
3. 后台管理员登录
4. 基础请求封装
5. 基础日志封装

### P1：紧接着完成

1. 微信端首页基础态
2. 个人中心登录态展示
3. 后台基础布局
4. 列表页和表格页骨架

### P2：后续再做

1. 公告中心
2. 消息中心
3. 营销体系
4. 分销体系
5. 复杂远程日志平台

---

## 3. 任务拆解

### 3.1 微信小程序端

- 登录注册页
- 手机号授权流程
- 统一请求封装
- 环境识别
- 日志封装
- 基础组件
  - `notice-bar`
  - `toast`
  - `loading`
  - `empty-state`
  - `confirm-dialog`

### 3.2 后端

- 微信登录接口
- 手机号绑定接口
- 当前登录态接口
- 访客身份接口
- 管理员登录接口
- 统一鉴权和角色判断

### 3.3 后台管理面板

- 登录页
- 布局骨架
- 请求封装
- 菜单权限
- 首个控制台首页

---

## 4. 里程碑

### 里程碑 1：身份底座

交付结果：

- 微信端可以登录 / 注册
- 后台可以登录
- 后端可以识别 `guest / user / admin`
- 请求链路可以带 token

### 里程碑 2：基础页面

交付结果：

- 微信端首页能正常进入
- 个人中心能显示登录态
- 后台能进入控制台首页

### 里程碑 3：基础能力

交付结果：

- 日志分级可用
- 环境区分可用
- 基础提示组件可复用

---

## 5. 任务依赖

先后顺序建议如下：

1. 环境和日志规范
2. 请求封装
3. 身份接口
4. 登录注册页
5. 后台登录页
6. 页面骨架
7. 业务页面

---

## 6. 风险点

- 微信手机号授权流程需要严格按官方能力实现
- 环境变量不要和微信运行态混淆
- 页面层不要直接写接口地址和日志规则
- 登录和注册必须合并，不要再拆成两套页面

---

## 7. 验收标准

- 微信端能完成登录 / 注册
- 后台能完成管理员登录
- 小程序能区分开发版、体验版、正式版
- 日志分级不会污染正式环境
- 请求层统一收口，不散落在页面里
