# 给其它 AI / 开发者的开发指令

你正在接手一个农仓电商项目，结构是：

```text
backend/ NestJS + Prisma
wxapp/miniprogram/ 微信小程序，包含 C端用户端 + B端商家端
admin/ Vue/Vite 平台管理后台
```

你的任务不是重做 UI，而是把现有页面功能和 API 接实，让最终农仓需求完整跑通。

## 一定要遵守

```text
1. 不要大改 UI，不要重写页面结构。
2. 只允许改 API 接入、字段映射、状态判断、跳转、错误提示。
3. 三身份固定：USER / MERCHANT / ADMIN。
4. 平台官方商家是 Merchant 数据，不是所有 MERCHANT 的替代。
5. 普通商家接口必须只查当前登录商家的数据。
6. 核心交易链路禁止 safeGet/safePost 静默 mock。
7. 父订单号只用于支付；子订单号用于订单详情、发货、物流、确认收货、评价、售后。
```

## 第一步必须先做

修后端身份边界：

```text
backend/src/common/services/platform-data.service.ts
```

新增：

```text
ensureCurrentMerchant(authUser)
```

并把所有普通 `/merchant/*` 业务改用它。

`ensurePlatformMerchant()` 只给：

```text
平台商品
平台客服
平台初始化
平台配置
```

## 第二步做订单状态机

状态必须固定：

```text
orderStatus: 1待接单 / 2已接单 / 3已完成 / 4已取消
payStatus: 0未支付 / 1已支付
deliveryStatus: 0未发货 / 1待发货 / 2已发货
refundStatus: 0无 / 1待商家 / 2处理中 / 3同意 / 4拒绝
```

前端所有按钮按这个判断。

## 第三步做订单字段合同

C端详情：

```http
GET /app/orders/:orderNo
```

B端详情：

```http
GET /merchant/orders/:orderNo
```

必须返回前端需要的商品、地址、金额、状态、时间字段。

## 第四步做核心闭环

必须按顺序通过：

```text
C端下单
→ 支付
→ B端接单
→ B端发货
→ C端看物流
→ C端确认收货
→ C端评价
→ B端回复评价
→ C端申请售后
→ B端处理售后
→ 平台查看/仲裁
```

## 第五步做营销闭环

```text
优惠券：领券 + 下单抵扣
秒杀：claim + flashPrice 创建订单
拼团：join + groupPrice 创建订单
促销价：SKU promotionPrice 服务端计算
```
