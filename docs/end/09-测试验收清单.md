# 测试验收清单

## 1. 启动验收

| 项 | 命令 | 通过标准 |
|---|---|---|
| 后端编译 | `cd backend && npm run build` | 无 TS 错误 |
| 后端启动 | `npm run start:dev` | 6002 或配置端口可访问 |
| Prisma | `npx prisma generate && npx prisma db push` | 无 schema 错误 |
| 小程序 TS | `cd wxapp && npx tsc --noEmit` | 无 TS 错误 |
| Admin | `cd admin && npm run build` | Vite 构建成功 |
| 健康检查 | `GET /health` | status=ok |

## 2. 身份验收

| 场景 | 通过标准 |
|---|---|
| 普通用户登录 | 返回 USER token，可访问 `/app/*` |
| 游客/匿名 | 可浏览公开商品，不可执行需要登录的交易动作 |
| 商家申请 | 生成 Merchant，状态待审核 |
| 平台审核商家 | Merchant.status=1，用户获得 MERCHANT 角色，钱包初始化 |
| 商家重新登录 | token role=MERCHANT |
| 平台管理员登录 | 返回 ADMIN token，只访问 `/admin/*` |
| 平台官方商家 | 存在 Merchant 记录，但不替代普通商家 |
| 多商家隔离 | 商家A看不到商家B订单/商品/钱包 |
