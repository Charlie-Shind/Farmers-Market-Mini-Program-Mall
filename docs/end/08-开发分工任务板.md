# 开发分工任务板

建议分 5 个 AI / 开发者并行，每个人只改自己的范围，避免互相覆盖。

## 分工总览

| 角色 | 负责范围 | 优先级 |
|---|---|---|
| 后端-A | 身份边界、订单状态机、订单详情、父子订单 | P0 |
| 后端-B | 评价、售后、秒杀、拼团、促销价、统计 | P0/P1 |
| C端开发 | 首页到订单、秒杀/拼团、评价、售后、聊天 API 接入 | P0/P1 |
| B端开发 | 商家订单、发货、售后、评价、财务、统计 API 接入 | P0/P1 |
| 平台后台开发/测试 | Admin build、平台配置、审核、Banner、公告、仲裁、全流程测试 | P0/P1 |

## 后端-A 任务

### A1. 修正商家身份边界

文件：

```text
backend/src/common/services/platform-data.service.ts
```

任务：

```text
1. 新增 ensureCurrentMerchant(authUser)。
2. 普通 /merchant/* 业务全部改用 ensureCurrentMerchant。
3. ensurePlatformMerchant 只保留给平台官方商家场景。
4. 确认普通商家不会读取平台官方商家订单/钱包/商品。
```

验收：

```text
两个不同商家登录，GET /merchant/orders 只能看到自己的订单。
```

### A2. 固定订单状态机

文件：

```text
backend/src/common/services/platform-data.service.ts
```

任务：

```text
1. 状态常量固定。
2. acceptOrder 只能处理已支付待接单。
3. shipOrder 只能处理已接单待发货。
4. confirmOrder 只能处理已发货。
5. refund 与订单状态互相约束。
```

验收：

```text
未支付不能接单；未接单不能发货；未发货不能确认收货。
```

### A3. 订单详情字段合同

文件：

```text
backend/src/common/services/platform-data.service.ts
```

任务：

```text
1. getOrderDetail 返回 C端字段。
2. getMerchantOrderDetail 返回 B端字段。
3. 日期统一 ISO 字符串。
4. addressSnapshot 兼容 receiverName/name/detailAddress/detail。
5. items 同时提供 orderItemId 与前端需要字段。
```

验收：

```text
C端订单详情、B端订单详情都能显示商品、金额、地址、按钮。
```

### A4. 父子订单号

任务：

```text
1. createOrder 普通购物车返回 childOrderNos。
2. 支付接口接收父订单号。
3. 详情/发货/售后/评价使用子订单号。
```
