# 需求覆盖矩阵

本矩阵按最终需求拆分。状态定义：

```text
已具备：已有页面 + API + 数据模型，允许少量字段修正。
需接实：页面已有，但仍有 mock/safeGet/字段错/状态错。
需补齐：缺核心接口或缺数据闭环。
可展示：可以做展示，不影响主闭环。
```

## 1. C 端需求覆盖

| 模块 | 需求 | 当前状态 | 必须处理 |
|---|---|---:|---|
| 首页 | 定位、搜索、消息入口 | 已具备 | API 地址正确即可 |
| 首页 | Banner、助农专题、新品预售 | 已具备 | 平台后台 Banner 配置接实 |
| 首页 | 秒杀/拼团/礼盒/产地直供快捷入口 | 已具备 | 秒杀/拼团必须真实跳转下单 |
| 推荐 | 智能品类推荐、热销榜 | 已具备 | 保证 `/app/home/hot-products` 返回真实商品 |
| 分类 | 一级类目、筛选 | 已具备 | 分类兜底可保留，主数据优先 API |
| 商品详情 | 图文、视频、多规格、服务说明 | 已具备 | 保证 SKU、服务标签、图片字段匹配 |
| 商品详情 | 溯源码扫码查看 | 已具备/需接实 | `/app/traces/:traceCode` 与详情 traceCode 对齐 |
| 商品详情 | 收藏、客服、加购、立即买 | 已具备 | 客服 sceneType/productId/orderNo 要正确 |
| 营销 | 优惠券、满减、新人券 | 已具备 | 领券、可用券、下单抵扣打通 |
| 营销 | 限时秒杀 | 需接实 | claim + 秒杀价订单 |
| 营销 | 邻里拼团 | 需接实 | join + 拼团价订单 |
| 营销 | 积分获得/兑换 | 部分具备 | 订单支付后积分增加、积分兑换记录 |
| 购物车 | 勾选、数量、凑单提示 | 已具备 | 凑单提示可展示，金额以后端为准 |
| 结算 | 地址、配送、券、积分、支付 | 需接实 | 父/子订单号、秒杀/拼团模式 |
| 订单 | 全状态查询 | 需接实 | 状态机统一 |
| 售后 | 退款申请 | 已具备/需验收 | `POST /app/refunds` 与订单状态约束 |
| 评价 | 提交评价 | 已具备/需验收 | `POST /app/orders/:orderNo/reviews` body 必须是 reviews 数组 |
| 客服 | 平台客服咨询 | 已具备 | support target 绑定平台官方 Merchant |

## 2. B 端商家需求覆盖

| 模块 | 需求 | 当前状态 | 必须处理 |
|---|---|---:|---|
| 店铺设置 | 店铺基础资料 | 已具备/需验收 | `/merchant/profile` |
| 资质管理 | 营业执照/产地资质 | 可展示/需补后端 | 非主闭环，允许 P2 |
| 商品管理 | 上下架、编辑 | 已具备 | `/merchant/products` 等接实 |
| SKU库存 | 库存增减 | 已具备 | `/merchant/skus/:skuId/stock` |
| 溯源资料 | 录入绑定 | 部分具备 | 商品编辑页字段与 ProductTrace 对齐 |
| 订单处理 | 接单 | 已具备 | 状态必须 PENDING+PAID |
| 物流发货 | 填公司和单号 | 已具备 | 必须先 accept 再 ship |
| 售后处理 | 退款同意/拒绝 | 已具备/需验收 | 列表、详情、处理全部接实 |

| 评价管理 | 查看评价并回复 | 已具备/需验收 | 回复 body 用 content |
| 消息中心 | 买家会话回复 | 已具备 | `/merchant/chats` |
| 经营统计 | 工作台和趋势 | 已具备/需验收 | `/merchant/workbench`、`/merchant/statistics/*` |

## 4. 不建议明天强补的需求

```text
1. 真实微信支付：保留模拟支付回调。
2. 真实顺丰物流：保留物流轨迹模拟。
3. 真实访客埋点：visitorCount 暂时为 0。
4. 社区团长分销完整结算：可作为扩展展示。
5. 复杂活动规则引擎：先用优惠券、秒杀、拼团、SKU促销价。
```
