# 湾源农仓最终开发交付包 README

本交付包用于给其它 AI / 开发者快速继续开发、联调和验收。文档目标不是重新设计 UI，而是把现有项目定稿为一个能完整跑通的三端农仓系统：

- C 端：普通用户微信小程序，负责浏览、领券、拼团/秒杀、下单、支付、售后、评价、客服。
- B 端：商家微信小程序后台，负责商品、库存、活动、订单、发货、售后、评价、财务、提现。
- 平台端：Web 管理后台，负责用户、商户、平台商品、Banner、活动、优惠券、订单、售后仲裁、财务审核、客服、数据看板。

## 当前项目结构

```text
farm/
├─ backend/                 NestJS + Prisma 后端
├─ wxapp/miniprogram/       微信小程序：C端 + B端商家端
├─ admin/                   Vue/Vite 平台管理后台
└─ docs/                    原项目文档，如存在
```

## 最重要的约束

1. 前端界面不能大改，只允许接 API、改字段映射、改状态判断、补跳转和错误提示。
2. 后端必须先定稿身份边界，尤其是“平台也是商家”的数据归属问题。
3. 主交易链路不能再依赖 mock / safeGet 兜底。
4. C 端普通用户、B 端普通商家、平台管理员必须能分别登录并各自看到自己的数据。
5. 平台官方商家是一个 Merchant 数据主体，但不是普通商家登录态的替代品。

## 当前检查结果摘要

- backend：`npm run build` 在给 `node_modules/.bin/*` 增加执行权限后可通过。
- wxapp：`npx tsc --noEmit` 可通过。
- admin：当前包内依赖缺 Rollup optional native 包，`npm run build` 失败。解决方式：删除 `admin/node_modules` 后重新 `npm install`。

```bash
cd admin
rm -rf node_modules package-lock.json
npm install
npm run build
```

## 开发优先级

```text
P0：身份边界、订单状态机、支付/订单/发货/物流/评价/售后闭环
P1：秒杀、拼团、优惠券、促销价、商家统计/财务闭环
P2：平台后台全量管理、Banner/公告/客服/售后仲裁/提现审核
P3：可展示但不强制真实的扩展：团长分销、复杂物流配置、复杂大屏
```
