# 02-2-接口-B端商家

> 面向农户/商家的经营后台接口说明，覆盖入驻、商品、订单、发货、活动、财务与通知。

## 0. 当前实现说明

- 路由前缀统一为 `/api/merchant`
- 默认要求商家角色登录，未登录用户不能访问写接口
- 商品、订单、活动、评价、退款已逐步从总服务拆出到独立 merchant service
- 草稿、详情、发布、删除等写操作仍沿用现有接口路径，不做破坏性变更

## 1. 商家入驻

### 1.1 提交申请

```http
POST /api/merchant/apply
```

- 创建商家主体
- 上传营业执照、产地资质等材料
- 默认进入待审核状态

## 2. 店铺与商品

### 2.1 商品发布

```http
POST /api/merchant/products
```

- 创建商品、SKU、图片、视频、溯源资料
- 通常先草稿或待审核，再由后台审核

### 2.2 商品上下架

```http
PATCH /api/merchant/products/{productId}/status
```

- 商品上架前需要审核通过

### 2.3 库存维护

```http
PATCH /api/merchant/skus/{skuId}/stock
```

- 仅允许修改属于当前商家的 SKU

### 2.4 商品草稿

```http
POST /api/merchant/products/drafts
GET /api/merchant/products/drafts
GET /api/merchant/products/drafts/{draftRef}
POST /api/merchant/products/drafts/{draftRef}
POST /api/merchant/products/drafts/{draftRef}/delete
```

- 草稿用于商品编辑过程中的临时保存
- `draftRef` 同时兼容草稿编号和数值 ID

## 3. 订单处理

### 3.1 商家订单列表

```http
GET /api/merchant/orders?status=4&page=1&pageSize=10
```

### 3.2 接单

```http
POST /api/merchant/orders/{orderNo}/accept
```

### 3.3 发货

```http
POST /api/merchant/orders/{orderNo}/ship
```

- 生成 `delivery_record`
- 更新订单为已发货

### 3.4 订单详情

```http
GET /api/merchant/orders/{orderNo}
```

- 展示订单主表、商品项、收货信息和售后状态
- 仅能查看当前商家的订单

## 4. 售后与财务

### 4.1 退款列表与处理

```http
GET /api/merchant/refunds
GET /api/merchant/refunds/{refundNo}
POST /api/merchant/refunds/{refundNo}/process
```

- 列表按当前商家过滤
- `process` 支持同意/拒绝退款
- 若退款成功，处理库存、积分、佣金回滚

### 4.2 钱包与提现

```http
GET /api/merchant/wallet
GET /api/merchant/finance/records
POST /api/merchant/withdraws
GET /api/merchant/withdraws
```

- 查询可提现余额
- 发起提现申请

### 4.3 评价管理

```http
GET /api/merchant/reviews
GET /api/merchant/reviews/summary
POST /api/merchant/reviews/{reviewId}/reply
```

- 支持按评分、回复状态筛选
- 回复评价仅限当前商家自己的评价数据

## 5. 活动与配送

### 5.1 活动管理

```http
GET /api/merchant/activities
POST /api/merchant/activities
PATCH /api/merchant/activities/{activityId}
DELETE /api/merchant/activities/{activityId}
GET /api/merchant/activities/{activityId}/detail
POST /api/merchant/activities/{activityId}/publish
POST /api/merchant/activities/{activityId}/pause
POST /api/merchant/activities/{activityId}/finish
POST /api/merchant/activities/{activityId}/copy
GET /api/merchant/activities/{activityId}/statistics
GET /api/merchant/activities/product-candidates
GET /api/merchant/activities/drafts
POST /api/merchant/activities/drafts
POST /api/merchant/activities/drafts/{draftId}/publish
DELETE /api/merchant/activities/drafts/{draftId}
```

- 活动列表与草稿保持分离
- 活动商品候选只返回当前商家可用商品

### 5.2 配送设置

```http
GET /api/merchant/delivery/settings
POST /api/merchant/delivery/settings
GET /api/merchant/delivery/templates
POST /api/merchant/delivery/templates
PUT /api/merchant/delivery/templates/{templateId}
DELETE /api/merchant/delivery/templates/{templateId}
```

- 配送模板和店铺配置仍保留现有路径

## 6. 工作台与通知

```http
GET /api/merchant/profile
GET /api/merchant/dashboard
GET /api/merchant/workbench
GET /api/merchant/statistics/overview
GET /api/merchant/statistics/trend
GET /api/merchant/notices
GET /api/merchant/logistics/companies
```

- 工作台与统计已拆到独立服务
- 通知、物流公司查询为只读接口
