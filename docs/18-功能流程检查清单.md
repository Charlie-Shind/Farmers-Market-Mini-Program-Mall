# 18-功能流程检查清单（基于 PRD 全量覆盖）

> 目的：作为项目质量总览与逐项排查依据，对照客户 PRD（已归档的本地备份）逐条验证功能是否实现。
> 排查对象：C 端小程序、B 端商家小程序、平台 Web 管理后台、对接后端。
> 本文档**只列检查项与判定标准**，不写实现方式；每条都能直接拿去做 smoke test 或人工验证。

---

## 0. 使用说明

### 0.1 检查项编号规则

- 章节号 + 子流程 + 顺序号，例如 `C-1-03` 表示 C 端第 1 大流程下第 3 项。
- 5 大流程前缀：`C` = C 端用户，`B` = B 端商家，`P` = 平台后台，`X` = 特色增值，`S` = 跨端系统/安全。

### 0.2 状态标记

每条检查项排查时打勾，4 种状态：

- ✅ **已实现**：可演示、可走通、有真实数据。
- ⚠ **部分实现**：页面/入口有但流程未闭环（如硬编码、空状态、无后端对接）。
- ❌ **未实现**：入口、页面、后端能力均不存在。
- 🚫 **不适用**：与当前阶段无关（如需明确才标记）。

### 0.3 排查顺序建议

1. 先做 `S` 类（鉴权、角色、协议、安全）——它是其他所有流程的前置。
2. 再按 `C → B → P → X` 顺序排查。
3. 每条都按"页面入口 → 接口请求 → 后端返回 → 数据落库 → 状态变更"5 步走查。

### 0.4 汇总表

排查完成后在文末【附录 A 汇总表】中按状态填写数量，作为本次回归结果。

### 0.5 本轮回归结果

- 已补齐页面与入口：活动专题页、浏览历史页、客服帮助页、统一商品列表页、物流详情页

- 已统一支付语义：未支付订单保留在列表中，按钮文案统一为”继续支付”，结算取消后回到订单详情

## S 跨端系统与安全检查

> 适用范围：所有端共用。**必须先通过再开始业务排查**。

| 编号 | 检查内容 | 检查方法 | 预期结果 |
| --- | --- | --- | --- |
| S-01 | C 端、B 端、平台后台三者登录态隔离 | 同一微信号分别尝试进入 C 端首页与平台后台；同账号用不同浏览器登录 | C 端登录不能进入平台后台，反之亦然；token 互不通用 |
| S-02 | C 端用户角色不可访问 B 端商家后台 | C 端登录态下访问商家中心 | 拒绝并提示"请使用商家账号" |
| S-03 | 平台后台登录失败/Token 失效自动回登录页 | 故意篡改 token 或后端返回 401 | 自动清缓存并跳转到登录页，不出现持续 Unauthorized |
| S-04 | 平台后台与小程序鉴权方案分离 | 阅读 auth 模块代码与小程序登录代码 | 不共用 session、不共用同一鉴权策略 |
| S-05 | 角色权限模型可区分：游客 / C 端用户 / 商家 / 平台运营 / 平台超管 | 查看后端 roles 模块与守卫 | 5 类角色在接口层有显式校验 |
| S-06 | 微信支付回调验签 | 模拟回调触发后，看后端是否校验签名 | 非法签名被拒，并记录日志 |
| S-07 | 退款/提现操作均需后端鉴权 | 直接 curl 后端退款/提现接口，伪造 userId | 越权调用被拒 |
| S-08 | 错误码/异常统一出口 | 触发任一异常（库存不足、订单状态非法等） | 返回结构与状态码符合接口契约 |
| S-09 | 文件上传走统一的 file 模块 | 上传商品图、资质图、头像 | 都通过 file 模块，URL 可访问 |
| S-10 | 二维码内容有版本与校验 | 扫描溯源码、团长码、订单核销码 | 解析后可正确路由到对应页面 |

## 附录 B 排查用例样例（用于 smoke test 落地）

> 写代码/做冒烟时，可直接照"前置 → 步骤 → 预期"格式展开。

**样例 1：完整下单流程（S → C-1 → C-2 → C-4 → C-6 → C-7 → C-8）**

- 前置：C 端已登录；有可用商品；后台已配置运费模板和支付。
- 步骤：
  1. 首页进入商品详情。
  2. 选择规格、加入购物车。
  3. 购物车勾选 → 进入结算。
  4. 选择地址、配送方式、券或积分。
  5. 微信支付（可走 mock 通道）。
  6. 支付成功后查看订单状态为"待发货"。
  7. 商家端接单 + 发货。
  8. C 端确认收货，订单切到"已完成"。
- 预期：全链路无报错，状态、库存、金额、积分、券状态全部正确。

**样例 2：售后退款（S → C-8 → B-3 → P-5）**

- 前置：已完成发货，未确认收货。
- 步骤：C 端申请退款 → 商家拒绝 → C 端申请平台介入 → 平台仲裁通过 → 资金原路退回。
- 预期：售后单状态变更与退款回调验签正常。

**样例 3：团长分销（X-01 ~ X-07）**

- 前置：已有团长账号、团长码。
- 步骤：团长分享码 → C 端扫码下单 → 选择自提 → 完成订单 → 团长端查询佣金。
- 预期：订单绑定团长关系，佣金按规则计算。
