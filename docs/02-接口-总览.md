## 1. 当前后端接口分层

按实际 NestJS 路由划分，后端接口主要分成 5 组：

| 分组 | 路由前缀 | 主要用途 |
| --- | --- | --- |
| 身份鉴权 | `/api/identity/auth` | 匿名态、微信登录、手机号绑定、个人资料 |
| C 端用户 | `/api/app/**` | 分类、商品、购物车、订单、消息、营销专区 |
| B 端商家 | `/api/merchant/**` | 商品、订单、活动、发货、店铺、通知、配送 |
| 平台后台 | `/api/admin/**` | 用户、商家、商品、活动、优惠券、Banner、物流、设置 |
| 文件上传 | `/api/files/upload` | 公共文件上传 |

### 1.1 当前业务落点

- `merchant` 侧的商品、订单、活动、评价、退款、工作台、统计已经拆分为多个独立 service
- `admin` 侧仍保留活动、商品审核、商户审核、退款仲裁、配置管理等后台能力
- `platform-data.service.ts` 现在更偏向“公共能力 + 未拆分遗留逻辑 + 平台级聚合”，不再作为唯一业务聚合层

## 4. 当前建议优先验证的接口

如果只是想快速确认后端“活着且主链路可读”，优先测：

- `GET /api/identity/auth/status`
- `GET /api/identity/auth/anonymous`
- `GET /api/app/categories`
- `GET /api/app/products?pageSize=3`
- `GET /api/app/quick/flash-sale/active`
- `GET /api/app/quick/group-buy/products?pageSize=3`

如果要进一步验证登录态读接口，再加：

- `GET /api/identity/auth/me`
- `GET /api/app/favorites`
- `GET /api/app/cart`
- `GET /api/app/orders`

## 5. 配套脚本

后端现有脚本分两类：

### 测试数据 / 场景注入

- `npm run scenario:test:all`
- `npm run scenario:test:order`
- `npm run scenario:test:message`
- `npm run scenario:payment:pending`
- `npm run scenario:payment:paid`
- `npm run scenario:flash:inject`
- `npm run scenario:flash:verify`
- `npm run scenario:group:inject`

### 接口烟测

- `npm run smoke:api`
- `npm run smoke:api -- --mode=auth`
