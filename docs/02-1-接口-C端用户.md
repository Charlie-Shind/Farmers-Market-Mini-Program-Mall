# 02-1-接口-C端用户

> 面向普通消费者的小程序接口说明，覆盖浏览、下单、支付、售后与个人中心。

## 0. 认证与登录

### 0.1 微信登录

```http
POST /api/auth/wechat/login
```

- 前端先调用 `wx.login()` 获取微信登录凭证
- 后端根据微信凭证换取微信身份信息
- 已存在账号则直接登录
- 不存在账号则创建用户并登录
- 若账号已被后台禁用，后端应返回统一的拒绝结果，前端不能继续沿用该账号的会话
- 返回的 `user` 中包含账号唯一标识 `accountNo`
- 返回的 `user` 也会带上基础会话摘要字段，便于前端直接展示昵称、头像、手机号、状态和最近登录时间
- `accountNo` 由后端生成，不替代数据库主键 `id`
- `accountNo` 为业务展示标识，格式遵循后端统一规则

### 0.2 手机号授权绑定

```http
POST /api/auth/wechat/phone-bind
```

- 前端通过微信手机号授权按钮触发
- 前端传递微信登录凭证 `loginCode` 和手机号授权凭证 `phoneCode`
- `phoneCode` 不是手机号本身，只是微信换取真实手机号的凭证
- 后端在服务端调用微信官方接口换取真实手机号，再写入用户资料
- 若当前账号已被禁用，绑定流程同样应被拦截，避免把手机号绑定到不可用账号
- 成功后返回的 `user` 同样包含 `accountNo` 和基础会话摘要字段
- 当微信配置缺失或上游换号失败时，后端返回通用错误，不暴露内部配置细节

### 0.3 账号唯一标识

- 该标识会随登录回包和 `GET /api/auth/me` 一起返回
- `accountNo` 是业务唯一号，按“日期 + 当日序号”生成
- 用户账号和管理员账号都使用独立前缀，避免混淆
- 对外展示和检索优先使用 `accountNo`，内部主键仍使用 `id`

### 0.4 当前登录态

```http
GET /api/auth/me
```

- 查询当前登录身份
- 用于区分游客、普通用户和商家身份
- 返回的 `user` 是可直接展示的用户摘要，至少包含 `accountNo`、`nickname`、`avatarUrl`、`mobile`、`status`、`lastLoginAt`
- `profile` 继续返回更完整的资料对象，供个人中心和资料页使用

### 0.5 访客身份

```http
POST /api/auth/guest
```

- 未登录用户可先获得临时身份 `guest`
- 每次签发的访客 token 都是独立会话，不应共享同一个匿名用户记录
- 仅用于浏览、临时购物车、临时收藏等弱身份场景
- 访客身份下的数据应默认隔离，不应和已登录用户合并展示

## 1. 首页

### 1.1 Banner

```http
GET /api/app/home/banners
```

- 返回首页活动 Banner
- 只展示启用且在有效期内的数据
- 按 `sortOrder` 升序返回，前端按数组做轮播渲染
- 可缓存
- 返回字段建议包含：
  - `id`
  - `title`
  - `imageUrl`
  - `linkType`
  - `linkId`
  - `startAt`
  - `endAt`
  - `sortOrder`
  - `status`

### 1.2 快捷入口

```http
GET /api/app/home/quick-entries
```

- 返回秒杀、拼团、礼盒、产地直供等入口

### 1.3 热销榜单

```http
GET /api/app/home/hot-products
```

- 按销量、浏览量、收藏量综合排序

## 2. 商品浏览

### 2.1 分类列表

```http
GET /api/app/categories
```

- 返回一级类目
- 供首页和分类页复用

### 2.2 商品列表

```http
GET /api/app/products?categoryId=1&originPlace=云南&isPreSale=false&page=1&pageSize=10
```

- 支持分类、产地、现货/预售等筛选
- 只返回已上架且审核通过的商品

### 2.3 商品详情

```http
GET /api/app/products/{productId}
```

- 返回商品信息、SKU、图片、视频、溯源信息
- 需要时可补充是否收藏状态

### 2.4 收藏商品

```http
POST /api/app/products/{productId}/favorite
DELETE /api/app/products/{productId}/favorite
```

- 收藏与取消收藏

## 3. 购物车

### 3.1 加入购物车

```http
POST /api/app/cart/items
```

- 校验登录
- 校验商品和 SKU 状态
- 检查库存
- 同一用户同一 SKU 只保留一条记录

### 3.2 购物车列表

```http
GET /api/app/cart
```

- 按商家分组返回

### 3.3 修改购物车

```http
PATCH /api/app/cart/items/{cartId}
DELETE /api/app/cart/items/{cartId}
```

- 修改数量、勾选状态、删除条目

## 4. 订单与支付

### 4.1 订单预结算

```http
POST /api/app/orders/preview
```

- 计算商品金额、运费、优惠券、积分抵扣

### 4.2 提交订单

```http
POST /api/app/orders
```

- 重新校验库存与价格
- 创建订单和订单明细快照
- 多商家时按商家拆单

### 4.3 微信支付

```http
POST /api/app/payments/wechat
POST /api/payments/wechat/callback
```

- 小程序端发起支付
- 微信回调由后端接收并做幂等处理

## 5. 订单中心

### 5.1 订单列表

```http
GET /api/app/orders?status=1&page=1&pageSize=10
```

### 5.2 取消订单

```http
POST /api/app/orders/{orderNo}/cancel
```

### 5.3 确认收货

```http
POST /api/app/orders/{orderNo}/confirm
```

## 6. 售后

### 6.1 申请售后

```http
POST /api/app/refunds
```

- 支持退款或退货退款
- 需要校验订单状态和可退金额

## 7. 营销与扩展

### 7.1 优惠券

```http
GET /api/app/coupons
POST /api/app/coupons/{couponId}/receive
```

### 7.2 积分

```http
GET /api/app/points/logs
```

### 7.3 团长相关

```http
POST /api/app/leaders/apply
POST /api/app/leaders/{leaderId}/bind
GET /api/leader/commissions
```
